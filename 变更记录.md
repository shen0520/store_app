### 1. 数据导入/导出功能

#### 需求
手机本地数据库相互独立，多个用户安装后无法共享商品数据。增加导入/导出功能，通过文件分享实现数据共享。

#### 新增文件

| 文件 | 功能 |
|------|------|
| `lib/services/export_service.dart` | 导出 JSON/CSV，调用系统分享面板（share_plus） |
| `lib/services/import_service.dart` | 选择文件（file_picker）、解析 JSON/CSV、合并到数据库 |
| `lib/pages/data_manage_page.dart` | 数据管理页面：显示商品数量、导出、导入、清空 |

#### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `pubspec.yaml` | 添加依赖 `file_picker: ^8.0.0+1` |
| `lib/database/db_helper.dart` | 新增方法：`importGoods()`（支持冲突策略）、`exportAllGoods()`、`clearAllGoods()` |
| `lib/pages/home_page.dart` | 首页新增「数据导入/导出」入口按钮 |

#### 功能特性

- **导出 JSON**：结构完整，含图片URL，可直接微信/QQ分享
- **导出 CSV**：Excel 可直接打开，方便给财务看
- **导入**：支持 JSON/CSV，三种冲突处理策略：
  - `skip` — 跳过重复（保留本地数据）
  - `overwrite` — 覆盖重复（用导入数据替换）
  - `merge` — 智能合并（保留 `update_time` 最新的）
- **清空数据**：一键清空，带二次确认弹窗

#### 使用流程

```
店主手机 → 导出 JSON → 微信发送 → 店员手机 → 导入 → 合并到本地数据库
```

### 2. 修复图片上传后不显示 Bug

#### 问题原因
- 用户从相册选择图片后，保存到数据库的是**本地文件路径**（如 `/storage/emulated/0/DCIM/xxx.jpg`）
- 但 `ScanPricePage` 和 `GoodsListPage` 里统一使用了 `Image.network()` 显示图片
- 本地路径不是网络 URL，`Image.network()` 无法加载，导致图片不显示

#### 修复方案

新增 `GoodsImage` 组件（`lib/widgets/goods_image.dart`）：
- 路径以 `/` 开头 → 用 `Image.file()` 显示本地文件
- 路径以 `http` 开头 → 用 `CachedNetworkImage` 显示网络图
- 其他/空值 → 显示「暂无图片」占位图

#### 修改文件

| 文件 | 修改 |
|------|------|
| `lib/widgets/goods_image.dart` | **新增**：智能图片显示组件 |
| `lib/pages/scan_price_page.dart` | `Image.network` → `GoodsImage` |
| `lib/pages/goods_list_page.dart` | `Image.network` → `GoodsImage` |
| `lib/pages/add_goods_page.dart` | `_buildImagePreview()` 简化为 `GoodsImage` |

### 3. 图片压缩

#### 问题
相册原图可能几MB一张，商品数量多后占用大量存储空间。

#### 修改
在 `AddGoodsPage._pickImage()` 中添加压缩参数：

```dart
picker.pickImage(
  source: ImageSource.gallery,
  maxWidth: 800,      // 最大宽度 800px
  maxHeight: 800,     // 最大高度 800px
  imageQuality: 80,   // JPEG 质量 80%
)
```

#### 压缩效果
- 原图 4MB → 压缩后约 100-300KB
- 1000 个商品 ≈ 100-300MB（可接受范围）

如需更省空间，可调低 `imageQuality`（如 60）或缩小 `maxWidth/maxHeight`（如 600）。
